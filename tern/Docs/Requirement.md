### 교수님 요구사항
#### 과제 요구
 가스환경 예측을 위한 6개 데이터로 3초 후, 9초 후, 30초 후, 60초 후, 120초 후를 예측할 수 있는 모델을 학습하여 보고서로 제출하시오. 포함할 내용 : 데이터셋 구성, 전처리 방법, 모델 구성, 학습 방법, 성능 평가

#### 녹취록
🎙️ 과제 관련 통합 녹취록
"자 이 데이터를 좀 소개를 해드리면요. 하나, 둘, 셋, 넷, 다섯, 여섯 개가 있습니다. 텀프로젝트 설명을 드리면, 센서들입니다. 이 배경을 좀 소개를 드리면, 가스 배관이 가스통에서 이렇게 쭉 연결이 되어져 있습니다. 특정한 포인트에서 엑셀로미터(진동), 가스 릭(누출), 프레셔(압력) 1/2, 템퍼러처(온도) 1/2 요렇게 6개의 센서로 주변에서 습득한 데이터입니다."

"목적으로 따지자면 이런 거예요. 과거에 이러한 데이터들이 쭉 있는데 향후에는 이게 어떻게 될 것인가. 즉, 몇 분 뒤에는 엑셀로미터, 가스 릭, 프레셔, 템퍼러처가 어떻게 변할 것인가를 예측하려고 하는 겁니다. 사고 발생 전에 온도가 높아지거나 압력이 떨어지는 현상들을 미리 몇 초라도 빨리 경보하기 위해 수집된 데이터입니다."

"이 데이터셋이 0000부터 0999까지 1000개의 파일(에피소드)로 있을 겁니다. 각각은 100밀리세크 단위로 수집된 300초(3000개)짜리 데이터입니다. 각각의 타임과 파일들은 사건적으로 상관이 없는 다른 에피소드입니다."

"저는 3초 이전에만 이 여섯 개의 센서값이 어떠한 값을 가질지만 알면 됩니다. 3초 후, 6초 후, 30초 후, 60초 후, 120초 후를 예측하시면 됩니다. 그럼 모델이 하나, 둘, 셋, 넷, 다섯 개 모델이겠네요."

"시계열 데이터입니다. LSTM, GRU, 시계열 컨볼루전(1D CNN), 아니면 컨볼루전에 LSTM 섞인 것 중 요 안에서만 사용하십시오. 개인적 요구사항입니다."

"해야겠다라고 하면은 10초 데이터 하고 정답은 3초 데이터 이렇게 하실 거잖아요? 쭉 이렇게 하다 보면은 마지막 3초 미래에 해당하는 것까지가 마지막 값이겠지? 그렇게 세트 하나. 그다음 다른 파일에서 또 라벨을 만드셔야지 됩니다. 보통 실수하는 게 다 CSV 파일이다 보니까 다 그냥 붙여놓고 에피소드가 바뀌었음에도 불구하고 윈도우를 잡는 경우가 있거든요. 그렇게 하시면 안 됩니다."

"여섯 개의 인풋 컬럼을 집어넣고 여섯 개의 출력을 내보내야 되지 않을까? 모델 하나씩 하기에는 너무 비효율적입니다. 원래 이 데이터 목적은 PC가 아니라 MCU 단에서 돌리는 걸 목적으로 했어서, 덴스 레이어(Dense Layer)로도 성능 나옵니다. 모델이 그렇게 크지 않아도 돼요."

"요구하는 거는 모델의 성능을 막 높이는 게 아니라, 그러한 데이터를 처리해 봤냐가 더 중요한 겁니다. 합리적이고 의미 있게 처리를 하십시오. 데이터가 메모리에 한 번에 다 안 올라가기 때문에 파이프라인 구성이 필요하기도 합니다."

"전처리 어떻게 하는지, 데이터셋 구성, 모델 구성 어떻게 했는지, 학습은 어떻게 시키고 성능 평가 어떻게 했는지 보고서에 적어주시면 됩니다. 제출일은 다음 주 목요일입니다. 보고서 진짜 잘 쓰셔야지 돼요."

📝 텀프로젝트(과제) 상세 설명 및 요구사항 분석
교수님의 녹취를 바탕으로 이번 텀프로젝트에서 달성해야 할 목표, 데이터 스펙, 모델링 조건, 주의사항을 구조화하여 정리했습니다.

1. 과제 개요 및 목표
배경: 가스 배관의 다중 센서 데이터를 분석하여 가스 누출 등의 사고 징후를 사전에 탐지하고 조기 경보를 울리기 위한 예측 시스템 개발.

목표: 과거 센서 데이터를 기반으로 미래 5개 시점(3초, 6초, 30초, 60초, 120초 후)의 6개 센서 값을 예측하는 딥러닝 모델 5개 구축.

2. 데이터셋 명세 (Data Specification)
특성 (Features): 총 6개 (진동, 가스 누출, 압력 1, 압력 2, 온도 1, 온도 2)

단위 및 규모: 100ms(0.1초) 단위로 측정. 1개 파일당 300초(3000개 행) 분량.

데이터 구성: 총 1,000개의 CSV 파일 (0000.csv ~ 0999.csv). 각 파일은 서로 독립적인 개별 '사고 에피소드'입니다. 센서마다 스케일(값의 범위)이 크게 다르므로 스케일링 전처리가 필수입니다.

3. 모델링 제약 및 권장 사항
입출력 구조: 6개의 변수를 한 번에 입력(Input) 받아 미래 시점의 6개 변수를 한 번에 출력(Output)하는 다변량 시계열 예측(Multivariate Time Series Forecasting) 구조로 설계해야 합니다. (센서별로 모델을 따로 만들지 말 것)

허용 알고리즘: 순환신경망(LSTM, GRU), 1차원 합성곱 신경망(1D CNN), 또는 이들의 결합 모델(ConvLSTM 등)로 제한됩니다.

경량화 (MCU 환경 고려): 애초에 소형 장비(MCU) 탑재를 목적으로 한 프로젝트이므로, 무겁고 복잡한 모델보다는 가벼운 모델(Dense Layer 위주의 단순한 구조 등)을 권장합니다.

4. 데이터 전처리 시 치명적 주의사항 (Hard Warning)
에피소드 분리 원칙: 1,000개의 CSV 파일은 독립적인 사건이므로, 모든 파일을 하나의 큰 데이터프레임으로 무작정 concat하여 슬라이딩 윈도우(Sliding Window)를 적용하면 절대 안 됩니다. 에피소드의 경계를 넘어가는 오염된 데이터 세트가 생성되기 때문입니다. 반드시 파일(에피소드) 단위로 윈도우를 분할한 뒤 합쳐야 합니다.

메모리 최적화 (파이프라인): 전체 데이터의 용량이 커서 한 번에 RAM에 올릴 수 없으므로, PyTorch의 Dataset/DataLoader나 TensorFlow의 tf.data 등을 활용하여 배치 단위로 데이터를 불러오는 파이프라인(Data Pipeline)을 구성해야 합니다.

5. 제출 및 평가 기준
기한: 다음 주 목요일

핵심 평가 지표: 모델의 단순 성능(정확도)보다는 '머신러닝 파이프라인의 구축 능력'과 '합리적인 데이터 전처리 및 탐색적 데이터 분석(EDA) 과정'을 훨씬 중요하게 평가합니다.
